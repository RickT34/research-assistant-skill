# Handbook Index

Use this file first to route common research-assistant requests to the most relevant handbook sections in [handbook.md](handbook.md).

## Core map

- User feels stuck, overwhelmed, or unsure whether they are making progress:
  - `§0 这份手册解决什么问题`
  - `§1 从“完成任务”切换到“推进问题”`
  - `§14 常见陷阱与处理方式`
- User needs to choose one weekly priority or recover project focus:
  - `§2 每周工作系统`
  - `§3.1 Boundary Card`
  - `§18.5 Weekly Report`
- User wants to define the current research question or decide whether an idea is worth doing:
  - `§3 拿到一个方向后，先写清边界`
  - `§10 从 failure mode 生成研究问题`
  - `§18.1 Problem Brief`
- User is reading papers and does not know what matters:
  - `§4 读论文：建立坐标系，不是被动读完`
  - `§15 向外看：在社区中定位你的发现`
- User needs to inspect a dataset or understand a benchmark:
  - `§5 先看数据，再跑模型`
- User is reproducing a paper or cannot match reported numbers:
  - `§6 复现：建立可信参照系`
  - `§8 Sanity Checks：全量实验前的体检`
  - `§14.5 结果和论文不一样`
- User cannot understand code, data flow, or generated code behavior:
  - `§7 读代码：沿着数据流，不沿着文件名`
  - `§8 Sanity Checks：全量实验前的体检`
  - `§14.4 看不懂代码`

- User has metrics but does not know what they mean:
  - `§8.6 指标不是能力本身`
  - `§9 Error Analysis：从分数进入问题本身`
- User needs failure analysis, slices, corner cases, or probing sets:
  - `§9 Error Analysis：从分数进入问题本身`
  - `§18.3 Error Analysis Report`
  - `§18.8 Toy / Proxy Test Card`
  - `§18.9 Probing Set Card`

  - `§10 从 failure mode 生成研究问题`
  - `§11 假设与实验设计：不要随机试错`
- User needs better project records, experiment logs, or repo hygiene:
  - `§12 项目管理与信息外化：建立 Project OS`

  - `§17 十二周上手路线`
