# Contributing

Thanks for helping improve this skill.

## What Makes A Good Change

- It helps users choose a concrete next action.
- It clarifies a repeated research failure mode.
- It improves section routing or citation accuracy.
- It adds a small reusable template, checklist, or example.
- It preserves the handbook's evidence-first, scope-bounded style.

## Handbook Changes

When editing `references/handbook.md`:

1. Make the smallest edit that solves the observed problem.
2. Prefer examples, trigger conditions, or edge-case notes over broad rewrites.
3. Update `references/handbook-index.md` if routing changes.
4. Add an entry to `references/handbook-revision-log.md`.

Revision log entries should include:

```md
## YYYY-MM-DD

- Trigger:
  - ...
- Change:
  - ...
- Rationale:
  - ...
```

## Style

- Keep advice concrete and action-oriented.
- Distinguish observed facts from hypotheses.
- Avoid making research sound more certain than the evidence supports.
- Cite handbook sections only when they materially shaped the answer.
- Do not include private project paths, unpublished student details, API keys, credentials, or local machine assumptions.
